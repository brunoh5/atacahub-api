# Módulo 3 — Clientes

## Objetivo

Este módulo é responsável pelo gerenciamento dos clientes do sistema, incluindo seus dados cadastrais, documentos, endereços, contatos e histórico de relacionamento.

O sistema deverá suportar tanto clientes Pessoa Física (PF) quanto Pessoa Jurídica (PJ), permitindo futuras integrações com módulos fiscais e financeiros.

---

## RF031 — Cadastro de Cliente

### Descrição

O sistema deverá permitir cadastrar novos clientes.

### Regras

- O cliente poderá ser Pessoa Física ou Pessoa Jurídica.
- O tipo de cliente não poderá ser alterado após o cadastro.
- O cliente deverá possuir status inicial **Ativo**.

### Dependências

RF004.

### Critérios de Aceite

- O cliente é cadastrado com sucesso.
- O tipo é armazenado corretamente.
- O cadastro é registrado na auditoria.

---

## RF032 — Consulta de Cliente

### Descrição

O sistema deverá permitir consultar os dados completos de um cliente.

### Dependências

RF031.

### Critérios de Aceite

- Todos os dados do cliente são retornados.
- Clientes inexistentes retornam erro.

---

## RF033 — Atualização de Cliente

### Descrição

O sistema deverá permitir atualizar os dados cadastrais do cliente.

### Regras

- Todas as alterações deverão ser auditadas.

### Dependências

RF032.

### Critérios de Aceite

- Os dados são atualizados corretamente.
- A auditoria registra as alterações.

---

## RF034 — Ativação e Desativação de Cliente

### Descrição

O sistema deverá permitir alterar o status do cliente.

### Regras

- Clientes inativos não poderão realizar novos pedidos.

### Dependências

RF033.

### Critérios de Aceite

- O status é alterado corretamente.

---

## RF035 — Exclusão Lógica de Cliente

### Descrição

O sistema deverá permitir realizar exclusão lógica de clientes.

RF034.

### Critérios de Aceite

### Descrição

### Regras

- Outro

### Dependências

RF031.

### Critérios de Aceite

- Um cliente pode possuir diversos endereços.
- Um endereço pode ser definido como padrão.

## RF039 — Cadastro de Documentos

### Descrição

O sistema deverá permitir armazenar documentos de identificação do cliente.

### Regras

Pessoa Física

- CPF
- RG (opcional)

Pessoa Jurídica

- CNPJ
- Inscrição Estadual (opcional)
- Inscrição Municipal (opcional)

### Dependências

RF031.

### Critérios de Aceite

- Os documentos são vinculados corretamente ao cliente.
- Não são permitidos documentos duplicados.
