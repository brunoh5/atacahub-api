# Módulo 2 — Controle de Acesso (RBAC) e Usuários

## Objetivo

Este módulo é responsável pelo gerenciamento dos usuários do sistema, papéis (Roles), permissões e políticas de acesso aos recursos da aplicação.

O modelo de autorização seguirá o padrão **Role-Based Access Control (RBAC)**, permitindo que um usuário possua um ou mais papéis e que cada papel contenha um conjunto de permissões.

---

## RF013 — Cadastro de Papéis (Roles)

### Descrição

O sistema deverá permitir o cadastro de novos papéis de acesso.

### Regras

- O nome do papel deverá ser único.
- O sistema deverá impedir a exclusão de papéis em uso.

### Dependências

Nenhuma.

### Critérios de Aceite

- O papel é criado com sucesso.
- Não permite nomes duplicados.
- O evento é registrado na auditoria.

---

## RF014 — Atualização de Papéis

### Descrição

O sistema deverá permitir alterar informações de um papel existente.

### Regras

- Alterações deverão ser registradas na auditoria.

### Dependências

RF013.

### Critérios de Aceite

- O papel é atualizado.
- A auditoria registra as alterações.

---

## RF015 — Exclusão de Papéis

### Descrição

O sistema deverá permitir excluir papéis que não estejam sendo utilizados.

### Regras

- Papéis vinculados a usuários não poderão ser removidos.

### Dependências

RF013.

### Critérios de Aceite

- Papéis sem utilização podem ser excluídos.
- O sistema impede exclusões inválidas.

---

## RF016 — Cadastro de Permissões

### Descrição

O sistema deverá permitir cadastrar permissões individuais.

### Regras

Cada permissão deverá possuir:

- Nome
- Código único
- Descrição

### Dependências

Nenhuma.

### Critérios de Aceite

- A permissão é criada.
- O código é único.

---

## RF017 — Associação de Permissões aos Papéis

### Descrição

O sistema deverá permitir associar permissões aos papéis cadastrados.

### Regras

- Uma permissão poderá pertencer a diversos papéis.

### Dependências

RF013

RF016

### Critérios de Aceite

- As permissões são associadas corretamente.
- A auditoria registra a alteração.

---

## RF018 — Associação de Papéis aos Usuários

### Descrição

O sistema deverá permitir associar um ou mais papéis aos usuários.

### Regras

- Um usuário poderá possuir múltiplos papéis.

### Dependências

RF013

RF001

### Critérios de Aceite

- Os papéis são vinculados ao usuário.
- As permissões tornam-se imediatamente válidas.

---

## RF019 — Consulta de Permissões

### Descrição

O sistema deverá permitir consultar todas as permissões efetivas de um usuário.

### Regras

- As permissões deverão considerar todos os papéis associados.

### Dependências

RF018

### Critérios de Aceite

- A consulta retorna todas as permissões do usuário.

---

## RF020 — Listagem de Usuários

### Descrição

O sistema deverá permitir listar usuários cadastrados.

### Regras

- A listagem deverá permitir paginação.
- A listagem deverá permitir ordenação.
- A listagem deverá permitir filtros.

### Dependências

RF001.

### Critérios de Aceite

- Os usuários são retornados corretamente.
- Os filtros funcionam corretamente.
- A paginação retorna os dados esperados.

---

## RF021 — Consulta de Usuário

### Descrição

O sistema deverá permitir consultar os detalhes de um usuário específico.

### Regras

- Apenas usuários autorizados poderão consultar outros usuários.

### Dependências

RF020.

### Critérios de Aceite

- Os dados são retornados corretamente.
- Usuários inexistentes retornam erro.

---

## RF022 — Atualização de Usuário

### Descrição

O sistema deverá permitir atualizar os dados cadastrais de um usuário.

### Regras

- O e-mail deverá permanecer único.
- Alterações deverão ser auditadas.

### Dependências

RF021.

### Critérios de Aceite

- Os dados são atualizados.
- O histórico da alteração é registrado.

---

## RF023 — Ativação e Desativação de Usuário

### Descrição

O sistema deverá permitir ativar ou desativar usuários.

### Regras

- Usuários desativados não poderão autenticar.

### Dependências

RF022.

### Critérios de Aceite

- O status é alterado corretamente.
- O login é bloqueado para usuários inativos.

---

## RF024 — Exclusão Lógica de Usuário

### Descrição

O sistema deverá permitir realizar exclusão lógica de usuários.

### Regras

- Os dados deverão permanecer armazenados para auditoria.
- O usuário não poderá mais acessar o sistema.

### Dependências

RF023.

### Critérios de Aceite

- O usuário deixa de aparecer nas listagens padrão.
- O histórico permanece disponível.

---

## RF025 — Perfil do Usuário

### Descrição

O sistema deverá permitir consultar o perfil do usuário autenticado.

### Dependências

RF004.

### Critérios de Aceite

- Os dados do usuário autenticado são retornados corretamente.

---

## RF026 — Atualização do Próprio Perfil

### Descrição

O sistema deverá permitir que o usuário altere seus próprios dados.

### Regras

- O usuário não poderá alterar papéis nem permissões.

### Dependências

RF025.

### Critérios de Aceite

- Apenas os dados permitidos são atualizados.

---

## RF027 — Pesquisa de Usuários

### Descrição

O sistema deverá permitir pesquisar usuários utilizando filtros.

### Regras

Filtros mínimos:

- Nome
- E-mail
- Status
- Papel
- Data de cadastro

### Dependências

RF020.

### Critérios de Aceite

- Todos os filtros funcionam corretamente.

---

## RF028 — Histórico de Alterações do Usuário

### Descrição

O sistema deverá manter um histórico das alterações realizadas em usuários.

### Regras

O histórico deverá armazenar:

- Usuário responsável
- Data
- Campo alterado
- Valor anterior
- Valor atual

### Dependências

RF022.

### Critérios de Aceite

- Todas as alterações ficam registradas.

---

## RF029 — Bloqueio Manual de Usuário

### Descrição

O sistema deverá permitir bloquear manualmente um usuário.

### Regras

- O bloqueio deverá encerrar todas as sessões ativas.

### Dependências

RF023.

### Critérios de Aceite

- O usuário perde acesso imediatamente.
- Todas as sessões são invalidadas.

---

## RF030 — Desbloqueio de Usuário

### Descrição

O sistema deverá permitir reativar usuários bloqueados manualmente.

### Dependências

RF029.

### Critérios de Aceite

- O usuário volta a acessar normalmente.
- O desbloqueio é registrado na auditoria.
