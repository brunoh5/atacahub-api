# Módulo 3 — Clientes

## Objetivo

Este módulo é responsável pelo gerenciamento do perfil comercial dos clientes da plataforma.

A autenticação, credenciais de acesso, gerenciamento de senhas, sessões e autenticação são responsabilidades do módulo IAM.

Este módulo gerencia os dados utilizados durante a experiência de compra, incluindo informações cadastrais, documentos, endereços, contatos, preferências e relacionamento comercial.

O sistema deverá suportar tanto clientes Pessoa Física (PF) quanto Pessoa Jurídica (PJ), permitindo futuras integrações com módulos fiscais, financeiros e de fidelidade.

---

## RF031 — Criação do Perfil Comercial do Cliente - ✅ Concluído

### Descrição

O sistema deverá criar automaticamente o perfil comercial de um cliente após a conclusão do cadastro de um usuário do tipo Cliente.

### Regras

- Todo Customer deverá estar vinculado exatamente a um User.
- Um User poderá possuir apenas um Customer.
- O perfil deverá ser criado na mesma transação do cadastro do usuário.
- O tipo do cliente (PF ou PJ) não poderá ser alterado posteriormente.

### Dependências

RF001.

### Critérios de Aceite

- O User é criado.
- O Customer é criado.
- Ambos permanecem vinculados.
- O cadastro é registrado na auditoria.

## RF060 — Conversão de Usuário em Cliente

### Descrição

O sistema deverá permitir que um usuário autenticado complete seu perfil comercial tornando-se um cliente apto a realizar compras.

### Regras

- O User deverá existir previamente.
- O Customer deverá ser criado apenas uma vez.
- O vínculo User ↔ Customer deverá ser permanente.

### Critérios de Aceite

- O perfil comercial é criado.
- O usuário passa a poder realizar compras.
