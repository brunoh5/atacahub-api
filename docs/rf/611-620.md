# Módulo 13 — Auditoria

# RF611 → RF620 — Registro e Rastreabilidade

## RF611 — Registro de Auditoria

### Objetivo

Registrar eventos relevantes realizados no sistema.

### Descrição

O sistema deverá registrar automaticamente todas as operações configuradas para auditoria, garantindo rastreabilidade das ações executadas pelos usuários e processos internos.

### Regras de Negócio

- Apenas eventos configurados deverão ser auditados.
- O registro deverá ser imutável.
- Nenhum registro poderá ser alterado manualmente.

### Dependências

- RF013

### Critérios de Aceite

- Os eventos são registrados corretamente.

---

## RF612 — Identificação do Usuário

### Objetivo

Associar cada registro ao usuário responsável.

### Descrição

Cada evento auditado deverá identificar o usuário ou serviço responsável pela operação.

### Regras de Negócio

O registro deverá conter:

- Usuário
- Perfil
- Serviço responsável (quando aplicável)

### Dependências

- RF611

### Critérios de Aceite

- O responsável pela ação é identificado corretamente.

---

## RF613 — Registro Temporal

### Objetivo

Registrar data e horário das operações.

### Descrição

Todo evento auditado deverá registrar a data e o horário exatos da execução.

### Regras de Negócio

- O horário deverá utilizar UTC internamente.

### Dependências

- RF611

### Critérios de Aceite

- Todos os registros possuem data e horário.

---

## RF614 — Identificação da Origem

### Objetivo

Registrar a origem da operação.

### Descrição

O sistema deverá registrar a origem de cada ação auditada.

### Regras de Negócio

A origem poderá ser:

- API
- Interface Administrativa
- Scheduler
- Processo interno
- Integração externa

### Dependências

- RF611

### Critérios de Aceite

- A origem é registrada corretamente.

---

## RF615 — Registro da Entidade

### Objetivo

Identificar qual entidade sofreu alteração.

### Descrição

O sistema deverá registrar a entidade afetada pela operação.

### Regras de Negócio

O registro deverá conter:

- Tipo da entidade
- Identificador
- Operação executada

### Dependências

- RF611

### Critérios de Aceite

- A entidade é identificada corretamente.

---

## RF616 — Valores Anteriores

### Objetivo

Registrar os valores anteriores da entidade.

### Descrição

Quando aplicável, o sistema deverá armazenar o estado anterior da entidade antes da alteração.

### Regras de Negócio

- Apenas campos configurados poderão ser armazenados.

### Dependências

- RF615

### Critérios de Aceite

- O estado anterior pode ser consultado.

---

## RF617 — Valores Atualizados

### Objetivo

Registrar o novo estado da entidade.

### Descrição

Após a operação, o sistema deverá registrar os novos valores da entidade.

### Regras de Negócio

- O registro deverá manter consistência com a operação executada.

### Dependências

- RF616

### Critérios de Aceite

- O estado atualizado é registrado corretamente.

---

## RF618 — Tipo da Operação

### Objetivo

Identificar a operação realizada.

### Descrição

Cada registro deverá informar qual operação foi executada.

### Regras de Negócio

Operações mínimas:

- CREATE
- UPDATE
- DELETE (lógico)
- LOGIN
- LOGOUT
- EXPORT
- IMPORT

### Dependências

- RF611

### Critérios de Aceite

- O tipo da operação é registrado corretamente.

---

## RF619 — Correlação de Eventos

### Objetivo

Relacionar eventos pertencentes à mesma operação.

### Descrição

O sistema deverá permitir correlacionar registros de auditoria gerados durante uma mesma transação ou fluxo de negócio.

### Regras de Negócio

- Cada operação deverá possuir um Correlation ID quando aplicável.

### Dependências

- RF611

### Critérios de Aceite

- Eventos relacionados podem ser consultados em conjunto.

---

## RF620 — Integridade dos Registros

### Objetivo

Garantir a integridade dos registros de auditoria.

### Descrição

Os registros de auditoria deverão permanecer íntegros durante todo o seu ciclo de vida.

### Regras de Negócio

- Os registros não poderão ser alterados.
- Alterações deverão gerar novos registros.

### Dependências

- RF611

### Critérios de Aceite

- A integridade dos registros é preservada.
